Javascript API Docs

View Methods

$.partial(name, data)

var str = $.partial("login", { username: 'bob', password: 'blah' });

console.log(str);// <input type='text' name='username' value='bob'/><input type='password' name='password' value='blah'>

Returns a string based on the partial template 'partial/.erb' with inserted. Default templating engine is underscore's builtin templates.

$.when(event, callback)

$("#submit").when("click", function(){
	alert('You submitted the form! Exciting!')
});

$.when() automatically registers/unregisters your event handlers on DOM elements. It is a proxy function for $.on() in jQuery. As long as you use $.when(), you don't have to worry about leaving event handlers attached when a view changes. DOM elements will never get held in memory by event handlers (an easy way to create memory leaks) or get double registered if a view loads twice.

URL Methods

$.go(controller, opts)

$.go("user_profile", { username : "bob" });

Switch pages to the new controller and optional parameters. This will change the current page hash, destroy the current view, and render the new view/controller.

$.goReplace(controller, opts, isQuiet)

$.goReplace("user_profile", { username : "bob" });

Occasionally you will have a page that updated its state but the view hasn't changed. You will still want to change the url to reflect the change to browser state (in case they reload the page). $.goReplace() provide the functionality to update the hash with the current view being destroy/rebuilt. The isQuiet parameter lets you decide if you want to add the new URL the browsers history.

$.link(controller, opts)

var url = $.link("user_profile", { username : "bob" });
console.log(url);// "#user-profile/bob"

Generates the proper URL for the controller and optional parameters. Returns a string that can be outputted into a template or manipulated programmatically. Based on route metadata provided to hasher.js via the routes.json file.
